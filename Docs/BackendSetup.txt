-----------------------------------------------------------
PART 1 — COMBAT SYSTEMS
-----------------------------------------------------------

1.1  Health.cs  (Combat/Scripts/Health.cs)
  Place on: every Player prefab, every Enemy prefab, every
            destructible object.
  Inspector settings:
    • maxHealth       — 100 for players, varies per enemy
    • isPlayer        — tick for all 5 player characters
                        (goes DOWNED instead of dying)
    • isRobotic       — tick for robotic enemies
                        (Defibrillator deals 60 extra dmg)

1.2  StatusEffectManager.cs  (Combat/Scripts/StatusEffectManager.cs)
  Place on: same GameObjects as Health.cs.
  No inspector setup needed — all code-driven.

1.3  EnemyAI.cs  (Combat/Scripts/EnemyAI.cs)
  Place on: every Enemy prefab.
  Inspector settings:
    • moveSpeed  = 3 (fast enemy) / 1.5 (slow)
    • aggroRange = 15
    • isElite    — tick for boss/elite enemies
                   (affects BountySystem CDR: 5s vs 2s)
    • isRobotic  — tick for robotic enemies
  Needs: NavMeshAgent OR Rigidbody on same GO.
  How it works: auto-finds nearest Player on Start.
    ThreatProtocol passive can call SetAggroTarget() to
    redirect all nearby enemies to the Guardian.

1.4  DeployableManager.cs  (Combat/Scripts/DeployableManager.cs)
  Place on: a persistent scene-root GameObject called
            "GameManagers" (DontDestroyOnLoad).
  This is a singleton — only ONE in the scene.
  No inspector setup. Used by:
    • ShockMineBehaviour
    • RestorationBeacon
    • BastionNode
    • PhaseRelayDeployable
    • ShadowRelayDeployable
    • TurretController (output multiplier)
    • AbilityCaster (SpawnDeployableAt helper)

1.5  SnapshotSystem.cs  (Combat/Scripts/SnapshotSystem.cs)
  Place on: same "GameManagers" object.
  Singleton. AbilityCaster calls Track() on Start for each
  player. System Rollback calls Rollback(5f).

-----------------------------------------------------------
PART 7 — HOOK CALL ORDER / INITIALIZATION
-----------------------------------------------------------

In your GameManager.Start() (or NetworkManager equivalent):

  1. Instantiate players
  2. For each player call:
       SnapshotSystem.Instance.Track(playerGO)
  3. After all players exist:
       FindObjectOfType<PassiveTriageLoop>()?.HookAllAllies()
  4. For each enemy spawn (in spawner script):
       FindObjectOfType<PassiveBountySystem>()?.HookEnemy(enemyGO)

PART 9 — ANIMATION SETUP
-----------------------------------------------------------

Animator Controller Setup (Unity 6.4)

Each player character needs an Animator with the following trigger parameters:
  • CastDamage  — damage abilities (primary attacks)
  • CastHeal    — healing abilities (support/restoration)
  • CastSupport — utility abilities (buffs, debuffs, control)

Animation clips available from brbmuffins Moves - Human Spellcasting Pack:
  Path: Assets/brbmuffins Moves/brbmuffins Human Animations/Animations/

  MALE and FEMALE variants available in:
    Idles/
      • CastingIdle01.fbx          — idle with wand/staff raised
    Combat/Spellcasting/
      • CastingEnter01.fbx         — enter casting stance
      • CastingExit01.fbx          — exit casting stance
      • CastingDamage01.fbx        — damage cast animation
    Combat/Spellcasting/MagicAttacks/
      • Directional/ variants      — attacks aimed in direction
        (Load, Cast versions available)
      • Omnidirectional/ variants  — attacks from caster position
        (Load, Cast versions available)

Animator States & Transitions

In your AnimatorController, create these states:
  1. Idle              (blend between CastingIdle + movement animations)
  2. CastDamage State  (triggered by CastDamage)
     ├─ animation: CastingDamage01.fbx or MagicAttack Cast clip
     ├─ duration: 0.8s typical
     └─ transition back to Idle on exit
  3. CastHeal State    (triggered by CastHeal)
     ├─ animation: CastingDamage01.fbx (can reuse, or use Omnidirectional_Load)
     ├─ duration: 1.0s typical (healing takes slightly longer)
     └─ transition back to Idle on exit
  4. CastSupport State (triggered by CastSupport)
     ├─ animation: CastingEnter01.fbx + CastingIdle01.fbx loop
     ├─ duration: variable by ability (0.5-2.0s)
     └─ transition back to Idle on exit

Timing Notes for AbilityCaster
  The CastAnimator.PlayCast(AbilityCategory) triggers the animator state.
  Ability damage hits on AbilityCaster.FinalizeCast() which can be synced to
  animation speed via AnimationEvent callbacks in Unity Editor:
    1. Select animation clip in inspector
    2. Add AnimationEvent at 60% animation completion
    3. Call a method to confirm damage is dealt

Class-Specific Animation Overrides (Optional)

For character-specific casting poses, create Animator Override Controllers:
  • Engineer:  use rifle/tech stance animations
  • Guardian:  use shield/heavy armor stance
  • Phaser:    use ethereal/phase animations
  • Medic:     use healing/support stance (arms open)
  • Wraith:    use stealth/shadow animations (low profile)

All can reuse the core CastDamage/CastHeal/CastSupport state machine with
different animation clips per class.

UI Implementation Notes

In AbilityBar.cs, map each ability to its icon:
  1. Store icon sprite reference in AbilityDef (ScriptableObject)
  2. In UI, assign icon to Image component: image.sprite = abilityDef.icon
  3. Recommended icon size: 64×64 or 128×128 pixels
  4. Set Image.preserveAspect = true in inspector
  5. Optionally: overlay cooldown radial progress bar on icon
     (use Mask component to reveal circular progress)

PART 11 — COMPILER / DEPENDENCY NOTES
-----------------------------------------------------------

Scripts that require other scripts on the same GameObject:
  AbilityCaster      → Health, (optional) PassivePhaseCharge,
                       PassiveBountySystem, CastAnimator
  SiegeModeHandler   → Health, PlayerMovement,
                       PassiveThreatProtocol
  KineticReversalHandler → Health
  TransferProtocolHandler → Health, LineRenderer
  IronTetherHandler  → Health, StatusEffectManager, LineRenderer
  StealthHandler     → Renderer(s) as children
  DashHandler        → Rigidbody OR CharacterController
  PassiveTriageLoop  → Health (self)
  PassiveThreatProtocol → Health
  PassivePhaseCharge  → AbilityCaster (finds via GetComponent)
  PassiveBountySystem → (hooks external enemy Health events)

Unity 6.4 API reminders:
  • FindObjectsByType<T>(FindObjectsSortMode.None) replaces
    FindObjectsOfType<T>() — already used throughout.
  • Physics.OverlapSphere returns Collider[] (not Collider2D[]).
  • rb.MovePosition requires Rigidbody.interpolation ≠ None
    for smooth visual results.
  • LineRenderer.material must be set (Sprites/Default used
    as fallback in all handlers).
